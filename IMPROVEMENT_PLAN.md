## 🔥 当前迭代任务（第一期）

### 任务 1：数据验证层
**目标**：在导入 Excel 时立即发现并报告数据问题

**待实现功能**：
- [x] 必填字段检查（姓名、人物ID）
- [x] 父母引用存在性检查
- [x] 循环引用检测（DFS 算法）
- [x] 多配偶数量提示（>3 个时警告）
- [x] 日期格式验证（出生日期 ≤ 死亡日期）

**验收标准**：
- ✅ 错误数据导入时，显示清晰的中文错误列表
- ✅ 定位到具体行号（如 "第 15 行：父亲ID 999 不存在"）
- ✅ 循环引用提示完整路径（如 "A → B → C → A"）

**实现细节**：
- 添加 `validateFamilyData()` 函数（返回错误数组）
- 添加 `detectCycles()` DFS 算法检测循环引用
- 在 `handleFileUpload()` 中调用验证，失败时阻止渲染
- 在 `#tree-container` 显示红色错误面板，包含"重新选择文件"按钮

**预估工时**：2h ✅ **已完成**
**完成时间**：2026-04-13 16:40
**代码变更**：
- `js/tree-generator.js`: +120 行（验证函数 + DFS 算法）
- `js/core.js`: +15 行（导入验证、错误显示、集成调用）

**测试建议**：
1. 准备测试 Excel 文件：
   - `test-orphan.xlsx`：包含不存在父母的引用
   - `test-cycle.xlsx`：A→B→C→A 循环引用
   - `test-multiple-spouses.xlsx`：>3 个配偶
2. 验证错误信息是否清晰、行号是否正确
3. 验证修复后的文件能否正常导入

### 任务 2：导出 SVG 格式
**目标**：在现有 PNG/PDF 基础上添加 SVG 导出按钮

**待实现功能**：
- [x] 实现 `exportAsSvg()` 函数
- [x] 使用 `XMLSerializer` 序列化 DOM
- [x] 添加 SVG 命名空间和必要属性
- [x] 内联 CSS 样式（确保独立可读）
- [x] 下载文件名：`家族族谱.svg`

**验收标准**：
- ✅ SVG 文件可在浏览器、Illustrator、Inkscape 中打开
- ✅ 文字保持可选中（非路径化）
- ✅ 文件大小 < PNG 的 50%

**实现细节**：
- 使用 `XMLSerializer` 将 DOM 转为 SVG 字符串
- 克隆 `#tree-content` 并添加内联样式
- 使用 `Blob` + `URL.createObjectURL` 触发下载
- 添加导出按钮到 HTML 的 `.buttons` 区域

**预估工时**：3h ✅ **已完成**
**完成时间**：2026-04-13 16:50
**代码变更**：
- `js/core.js`: +45 行（`exportAsSvg()` 函数 + 按钮绑定）
- `index.html`: 添加 `<button id="exportSvg">`

**测试建议**：
1. 生成一个中等复杂度的族谱（50-100 人）
2. 点击"导出为SVG"按钮
3. 检查文件大小（应为 PNG 的 20-40%）
4. 用浏览器打开 SVG，确认文字可选中
5. 用 Illustrator/Inkscape 打开，确认图层完整

**已知限制**：
- SVG 仅支持矢量元素，CSS 复杂样式可能略有差异
- 阴影、圆角等效果在部分旧版矢量软件中可能不支持
